Imports order

When importing modules, it's important to follow a consistent order to improve readability and maintainability of your code. A common convention is to group imports into three categories:

  1. External libraries: These are third-party modules that you install via package managers like npm or yarn. They should be imported first.
  2. Internal modules: These are modules that you have created within your project. They should be imported after external libraries.
  3. Stylesheets: If you're importing CSS or other stylesheets, they should be imported last.

Following this order helps to clearly distinguish between different types of dependencies and makes it easier for developers to understand the structure of the codebase.

the var keyword

The var keyword is used to declare variables in JavaScript. However, it has some quirks and limitations compared to the modern let and const keywords.

One of the main issues with var is that it is function-scoped, which means that variables declared with var are accessible throughout the entire function in which they are declared, regardless of block scope. This can lead to unintended consequences and bugs in your code.

In contrast, let and const are block-scoped, meaning that they are only accessible within the block in which they are declared. This helps to prevent accidental variable shadowing and makes it easier to manage variable scope.

In general, it's recommended to avoid using var in modern JavaScript development and instead use let and const for better code clarity and maintainability.
