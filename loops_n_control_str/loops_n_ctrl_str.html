Passing arguments by Value and reference

In JavaScript, the behavior of "passing" depends on whether you are using primitive or non-primitive data types. Technically, JavaScript is always pass-by-value, but the "value" passed for objects and arrays is the reference to that data in memory

There is quick comparison in the next table

feature Primitives (pass by value) Non-Primitives (pass by reference*)
Data types Numbers, Strings, booleans, nulls, undefined Objects, Arrays, functions
how it works Creates a copy of the actual data Passes a copy of the reference(Adress)
Effect of changes Changes inside a functions, do not affect the original Modify the properties affects the original data
Reassignment Reassigning the variable replaces the value Reassigning the variable replaces the link to the original

The if else

The if...else statement is a fundamental control flow structure in programming that executes a block of code if a condition is true, and an alternative block if it is false. It acts as a decision-making tool, allowing programs to follow different paths based on runtime evaluation. Here is an example

                
                    if ( [Expression to evaluate] ){
                        [... Action to execute ...]
                    } else {
                        [... Do another thing instead ...]
                    }

Boolean logic

A boolean function is a mathematical function that maps arguments to a value, where the allowable values of range (the function arguments) and domain (the function value) are just one of two values— true and false (or 0 and 1). The study of boolean functions is known as Boolean logic.

The ternary operator

The ternary operator in JavaScript, also known as the conditional operator, is a shorthand for writing if...else statements in a single line. It is called "ternary" because it is the only JavaScript operator that takes three operands
