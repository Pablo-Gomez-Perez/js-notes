Passing arguments by Value and reference

In JavaScript, the behavior of "passing" depends on whether you are using primitive or non-primitive data types. Technically, JavaScript is always pass-by-value, but the "value" passed for objects and arrays is the reference to that data in memory

There is quick comparison in the next table

feature Primitives (pass by value) Non-Primitives (pass by reference*)
Data types Numbers, Strings, booleans, nulls, undefined Objects, Arrays, functions
how it works Creates a copy of the actual data Passes a copy of the reference(Adress)
Effect of changes Changes inside a functions, do not affect the original Modify the properties affects the original data
Reassignment Reassigning the variable replaces the value Reassigning the variable replaces the link to the original
